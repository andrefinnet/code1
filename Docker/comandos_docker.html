-Rodando o Ubuntu:

  1. 01 - $docker run -it ubuntu bash
  2. -------------------------------
  3. Fixando o conhecimento:
  4. 02 - $docker run -it ubuntu echo Aprendendo Docker
  5. -Esse Primeiro comando vai iniciar um container em modo interativo
    e com um terminal anexado baseado na ultima imagem do ubuntu
    disponivel!.
  6. --------------------------------
  7. 03 - $docker run -it ubuntu sh -c 'echo aprendendo docker
  8. -Esse segundo comando vai ter a mesma saida, porém, da maneira que eexecutado
    poderiamos ter concatenados alguns outro comandos dentro do
    parametro -c ("comando"), ja que neste caso o processo-base para o container não foi o comando echo, mais sim o sh
  9. --------------------------------
  10. 04 - $docker run -i ubuntu ping -c 7 google.com

  11. -No Terceiro comando ping será executado sete
    vezes dentro do container, e a saida e mostrada na tela,
    quando terminar o container e encerrado.
  12. ------------------------------
  13. - 05 - $docker run -d ubuntu ping google.com
  14. -No Quarto Comando, o mesmo ping e executado,
    mais a flag -d detached foi passada. Então, no output aparece apenas o ID longo do conteiner.
    Enquanto o ping esta rodando mesmo que sem mostrar o output, o contêiner permanece de pé. Ao terminar o ping
    o contêiner e encerrado, como está em segundo plano (background), nem percebemos.
  15. -------------------------------

  16. Principais comandos do Docker:

  17. 06 - docker info:
  18. Quando terminamos de subir o Docker Engine testamos
    com as principais informações usando o $docker info
    (informações sobre o docker host no momento).
  19. -------------------------------
  20. - 07 - docker run:
  21. Este e um dos mais importantes comandos do Docker.
    Ele executa um contêiner com base em sua imagem e tem diversos
    parametros possiveis para combinações e configurações.
  22. -------------------------------
  23. - 08 - docker ps:
  24. Este provavelmente e o comando que mais será utilizado durante nossa aventura com o docker.
    Ele serve para mostrar detalhes sobre os containers.
  25. -----------------------------------
  26. - 09 - docker images:
  27. Este comando e muito parecido com docker ps,
    serve para listar as imagens que temos localmente em nosso Docker Host/
  28. -------------------------------------------
  29. - 10 - docker attach:
  30. Esse conado foi muito utilizado antes de inventarem o docker exec.
    Serve para anexar um contêiner pelo nome ou pela ID para assumirmos o controle
    sobre ele.
  31. -------------------------------------------
  32. - 11 - docker exec
  33. É um comando relativamente novo, que existe desde
    adistribuição da versão 13.0 em outubro de 2014.
  34. Este comando adicionaou um processo a mais a um contêoner que esteja rodando.
